Create a Firebase Project

Required
  1. Go to console.firebase.google.com
  2. Click "Create a project" — name it tradecommand-pro
  3. Disable Google Analytics (optional, not needed)
  4. Once created, click the Web icon (</>) to register a web app
  5. Name it TradeCommand Pro and click Register
  6. You'll see a config object — copy these values:
const firebaseConfig = {
  apiKey: "AIzaSy...",
  authDomain: "tradecommand-pro.firebaseapp.com",
  projectId: "tradecommand-pro",
  storageBucket: "tradecommand-pro.appspot.com",
  messagingSenderId: "123456789",
  appId: "1:123456789:web:abc123"
};

Paste these values into 3 files: dashboard.html, admin.html, and business.html. Search for YOUR_FIREBASE_API_KEY and replace each placeholder.

2

Enable Authentication

Required
  1. In Firebase Console, go to Authentication → Sign-in method
  2. Enable Email/Password — toggle it on, click Save
  3. Enable Google — select your project support email, click Save
  4. Go to Settings → Authorized domains
  5. Add maxmcdonough.github.io to the authorized domains list
3

Set Up Firestore Database

Required
  1. In Firebase Console, go to Firestore Database → Create database
  2. Choose "Start in test mode" (you'll lock it down later)
  3. Select region us-west1 (closest to Vancouver)
  4. Click Enable
⚠️ Test mode allows open access for 30 days. Before going live, update your Firestore rules (see below).

Production security rules:

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
    match /{document=**} {
      allow read, write: if request.auth != null
        && request.auth.token.email == 'max.mcdonough123@gmail.com';
    }
    match /activity/{doc} {
      allow create: if request.auth != null;
      allow read: if request.auth.token.email == 'max.mcdonough123@gmail.com';
    }
  }
}
4

Set Up Stripe Payments

Required for Pro subscriptions
  1. Go to dashboard.stripe.com and create an account
  2. Go to Products → Add product
  3. Create "TC Pro Monthly" — Price: $12.99 CAD, Recurring monthly
  4. Create "TC Pro Annual" — Price: $109.99 CAD, Recurring yearly
  5. For each product, click "Create payment link"
  6. Set the Success URL to: https://maxmcdonough.github.io/tradecommand-pro/dashboard.html?payment=success
  7. Set the Cancel URL to: https://maxmcdonough.github.io/tradecommand-pro/dashboard.html?payment=cancel
  8. Copy each payment link URL
  9. Go to Settings → Customer portal, enable it, copy the portal link
  10. Paste all three URLs into dashboard.html in the STRIPE_CONFIG section
const STRIPE_CONFIG = {
  monthlyLink: 'https://buy.stripe.com/your_monthly_link',
  annualLink: 'https://buy.stripe.com/your_annual_link',
  customerPortal: 'https://billing.stripe.com/p/login/your_portal',
  monthlyPrice: 12.99,
  annualPrice: 109.99,
  currency: 'CAD'
};

Stripe Webhooks (Optional)

Advanced

For automatic plan updates when payments succeed, set up a Firebase Cloud Function webhook. Without this, plans update client-side on redirect and can be set manually in the Admin panel.

  1. Install Firebase CLI: npm install -g firebase-tools
  2. Run firebase init functions in your project folder
  3. Create a webhook that listens for checkout.session.completed
  4. The webhook updates the user's Firestore doc with plan: 'pro_monthly' or pro_annual'
  5. Add the webhook URL in Stripe → Developers → Webhooks
